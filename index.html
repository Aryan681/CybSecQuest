Cyber Security FAQs

2 Marks

1. What is a cyber threat?
A cyber threat is a potential malicious act that seeks to damage data, steal data, or disrupt digital life in general. Examples include malware, phishing, and DDoS attacks.
2. Enumerate the needs for a nodal authority.
A nodal authority is required to coordinate cyber security efforts, manage incident responses, ensure compliance with regulations, and provide a central point for communication and guidance.
3. Define weak authentication.
Weak authentication refers to inadequate methods of verifying a user's identity, often involving easily guessable passwords or insufficient security measures, making it vulnerable to breaches.
4. State denial of service.
Denial of Service (DoS) is an attack intended to shut down a machine or network, making it inaccessible to its intended users by overwhelming it with a flood of illegitimate requests.
5. What is identity management?
Identity management involves the processes and technologies used to manage and secure individuals' access to resources and data, ensuring that only authorized users can access specific information or systems.
6. Give the uses of authorization patterns.
Authorization patterns are used to control access to resources, enforce security policies, and ensure that users have the appropriate permissions for their roles and responsibilities.
7. Specify the cause for abuse of privileges.
Abuse of privileges occurs when users with elevated permissions misuse their access to resources, either maliciously or negligently, leading to data breaches or unauthorized actions.
8. Mention the validation steps of system integrity.
Validation steps for system integrity include regular audits, checksum verification, intrusion detection, configuration management, and monitoring for unauthorized changes.
9. What is a digital signature?
A digital signature is an electronic form of a signature that uses cryptographic techniques to verify the authenticity and integrity of a digital message or document.
10. List any two properties of asymmetric key cryptography.
  • Public and Private Keys: Uses a pair of keys where one encrypts (public key) and the other decrypts (private key).
  • Confidentiality and Authentication: Ensures that only the intended recipient can decrypt the message and verifies the sender's identity.
11. Who governs the internet?
The internet is governed by a decentralized network of international organizations, including ICANN (Internet Corporation for Assigned Names and Numbers), IETF (Internet Engineering Task Force), and various national regulatory bodies.
12. Enumerate the vulnerabilities in software.
Common vulnerabilities include buffer overflows, SQL injection, cross-site scripting (XSS), and insecure configurations.
13. Define system integrity.
System integrity refers to the state of a system where it is performing its intended functions without being altered or tampered with, ensuring reliability and correctness.
14. What is an insider threat?
An insider threat involves a risk posed by someone within an organization, such as an employee or contractor, who may misuse their authorized access to harm the organization.
15. List the basic SOAP Services.
Basic SOAP services include service description via WSDL (Web Services Description Language), message format using XML, and communication protocols like HTTP/HTTPS.
16. Give the uses of SSL and TLS.
SSL (Secure Sockets Layer) and TLS (Transport Layer Security) are used to encrypt data transmitted over the internet, ensuring secure communications and protecting data from interception.
17. Specify the securities at network layer.
Network layer security includes IPsec for securing IP communications, VPNs for secure remote access, and firewalls for filtering unauthorized traffic.
18. Mention the basic security for HTTP applications.
Basic security for HTTP applications includes HTTPS for encrypted communication, input validation to prevent injections, and authentication mechanisms like OAuth.
19. What is SQL injection?
SQL injection is an attack technique where malicious SQL code is inserted into an input field for execution, allowing attackers to manipulate or access the database.
20. List any two cryptographic techniques.
  • Symmetric Encryption: Uses the same key for encryption and decryption, e.g., AES.
  • Asymmetric Encryption: Uses a pair of keys (public and private) for encryption and decryption, e.g., RSA.

7 Marks

1. Write short note on denial of service filters.
Denial of Service (DoS) filters are mechanisms designed to detect and mitigate DoS attacks, which aim to overwhelm a network or service with excessive traffic. These filters analyze incoming traffic to identify patterns indicative of an attack and then block or limit traffic from suspicious sources. Techniques include rate limiting, IP blacklisting, anomaly detection, and leveraging cloud-based mitigation services. Effective DoS filters ensure service availability and maintain performance during an attack.
2. What is ethical hacking? Explain in detail.
Ethical hacking involves authorized attempts to breach an organization's security systems to identify vulnerabilities before malicious hackers can exploit them. Ethical hackers, also known as white-hat hackers, follow a structured approach: reconnaissance, scanning, gaining access, maintaining access, and covering tracks. They use various tools and techniques to simulate attacks, uncover security weaknesses, and provide recommendations for remediation. Ethical hacking helps organizations strengthen their defenses, comply with regulations, and protect sensitive data.
3. Describe network-based intrusion detection systems.
Network-Based Intrusion Detection Systems (NIDS) monitor network traffic for suspicious activity and potential threats. NIDS analyze packet data and compare it against known attack signatures and anomaly behaviors. Deployed at strategic points in the network, NIDS can detect a range of threats, including malware, unauthorized access, and Denial of Service (DoS) attacks. When a threat is detected, NIDS generate alerts for administrators to investigate and respond. Advanced NIDS may also include capabilities for intrusion prevention, automatically blocking identified threats.
4. List various cyber security considerations and challenges.
Considerations:
  • Data Protection: Ensuring confidentiality, integrity, and availability of data.
  • Access Control: Implementing robust authentication and authorization mechanisms.
  • Incident Response: Developing and maintaining an effective response plan.
  • Compliance: Adhering to relevant laws and industry regulations.
Challenges:
  • Evolving Threats: Keeping pace with sophisticated and emerging cyber threats.
  • Resource Constraints: Balancing budget and staffing limitations.
  • User Behavior: Mitigating risks associated with human error and insider threats.
  • Technology Integration: Ensuring seamless integration of security solutions with existing infrastructure.
5. What is VPN Security Protocols? Describe in detail.
VPN (Virtual Private Network) security protocols establish secure communication channels over public networks. Key protocols include:
  • IPsec (Internet Protocol Security): Provides end-to-end encryption, data integrity, and authentication for IP packets. It operates in two modes: transport mode (encrypts only the payload) and tunnel mode (encrypts the entire packet).
  • SSL/TLS (Secure Sockets Layer/Transport Layer Security): Used primarily for secure web browsing, these protocols encrypt data at the transport layer, ensuring secure communication between client and server.
  • PPTP (Point-to-Point Tunneling Protocol): An older protocol that encapsulates PPP (Point-to-Point Protocol) frames, providing basic encryption and authentication.
  • L2TP (Layer 2 Tunneling Protocol): Often combined with IPsec, L2TP provides a secure tunnel for data transmission.
VPN security protocols are essential for protecting data privacy, ensuring secure remote access, and maintaining data integrity over untrusted networks.
6. Distinguish firewalls and intrusion detection systems.
Firewalls:
  • Purpose: Control incoming and outgoing network traffic based on predetermined security rules.
  • Operation: Filters traffic at the network perimeter, blocking unauthorized access.
  • Types: Packet-filtering, stateful inspection, and application-level gateways.
Intrusion Detection Systems (IDS):
  • Purpose: Monitor network or system activity for malicious behavior and policy violations.
  • Operation: Analyzes traffic patterns, detects anomalies, and alerts administrators.
  • Types: Network-based (NIDS) and host-based (HIDS).
Differences:
  • Firewalls are proactive, blocking unauthorized access, while IDS are reactive, detecting and alerting on suspicious activities.
  • Firewalls operate at the network perimeter; IDS can operate both at the network and host level.
7. List the various host-based intrusion prevention systems.
Host-Based Intrusion Prevention Systems (HIPS) are installed on individual devices to prevent malicious activities. Key examples include:
  • McAfee Host Intrusion Prevention: Protects against known and zero-day threats with signature-based and behavioral analysis.
  • Symantec Endpoint Protection: Offers comprehensive security with firewall, antivirus, and intrusion prevention features.
  • Trend Micro Deep Security: Provides advanced threat protection, including anti-malware, web reputation, and firewall controls.
  • Carbon Black: Utilizes endpoint detection and response (EDR) capabilities to identify and stop attacks in real-time.
8. Write short note on internet governance.
Internet governance involves the development and application of shared principles, norms, rules, and decision-making procedures that shape the evolution and use of the internet. Key organizations include:
  • ICANN (Internet Corporation for Assigned Names and Numbers): Manages domain name systems (DNS) and IP address allocation.
  • IETF (Internet Engineering Task Force): Develops and promotes internet standards.
  • ISOC (Internet Society): Advocates for internet policies and education.
  • W3C (World Wide Web Consortium): Develops web standards.
Internet governance is a collaborative effort involving governments, private sector, civil society, and international organizations to ensure the internet remains open, secure, and accessible.
9. What is deception? Explain in detail.
Deception in cyber security involves creating false information or environments to mislead attackers, divert their efforts, and gain insights into their techniques. Key components include:
  • Honeypots: Decoy systems designed to attract attackers, allowing monitoring and analysis of their actions.
  • Honeytokens: Fake data elements, such as credentials or files, that alert administrators when accessed.
  • Deception grids: Network architectures that create a maze of false paths and systems to confuse attackers.
Deception techniques enhance security by wasting attackers' time, gathering intelligence, and identifying potential breaches before they affect real assets.
10. Describe the basic security for HTTP applications.
Basic security measures for HTTP applications include:
  • HTTPS: Encrypts data transmitted between the client and server using SSL/TLS, protecting against eavesdropping and man-in-the-middle attacks.
  • Input Validation: Ensures that data input by users is properly checked and sanitized to prevent SQL injection and cross-site scripting (XSS).
  • Authentication and Authorization: Implements robust mechanisms to verify user identities and enforce access controls, such as OAuth, JWT, and multi-factor authentication (MFA).
  • Security Headers: Adds HTTP headers like Content Security Policy (CSP), X-Content-Type-Options, and Strict-Transport-Security to mitigate various attacks.
11. List various intrusion detection and prevention techniques.
Intrusion Detection and Prevention Systems (IDPS) use several techniques:
  • Signature-Based Detection: Identifies known threats by comparing network traffic to a database of attack patterns.
  • Anomaly-Based Detection: Detects deviations from normal behavior, flagging unusual activities as potential threats.
  • Behavioral Analysis: Monitors user and system behavior to identify suspicious activities.
  • Heuristic-Based Detection: Uses algorithms to detect unknown threats based on patterns and behaviors.
  • Network Behavior Analysis (NBA): Examines network traffic to identify unusual patterns that may indicate an attack.
12. What is network session analysis? Describe in detail.
Network session analysis involves examining the details of network sessions to monitor and understand communication patterns. It helps in:
  • Identifying Anomalies: Detects unusual session characteristics, such as unexpected duration, volume, or frequency, which may indicate a security threat.
  • Traffic Analysis: Analyzes the flow of data to and from network endpoints to identify potential security incidents or policy violations.
  • Protocol Analysis: Examines the use of network protocols to ensure compliance with security policies and to detect misuse or exploitation.
  • Forensic Investigations: Provides detailed session logs that can be used to investigate and reconstruct security incidents, aiding in the identification of attackers and their methods.
13. Explain the different types of cryptographic techniques.
Cryptographic techniques ensure data security through encryption and decryption processes. Key types include:
  • Symmetric Cryptography: Uses the same key for both encryption and decryption. Examples include AES (Advanced Encryption Standard) and DES (Data Encryption Standard).
  • Asymmetric Cryptography: Uses a pair of keys – public key for encryption and private key for decryption. Examples include RSA (Rivest-Shamir-Adleman) and ECC (Elliptic Curve Cryptography).
  • Hash Functions: Converts data into a fixed-size hash value, which is unique to the original data. Examples include SHA-256 (Secure Hash Algorithm) and MD5 (Message Digest Algorithm).
  • Digital Signatures: Provides data integrity and authenticity by using asymmetric cryptography to sign and verify messages.
14. List the uses of Pretty Good Privacy (PGP).
Pretty Good Privacy (PGP) is used for:
  • Email Encryption: Secures the content of emails, ensuring confidentiality and privacy.
  • Digital Signatures: Verifies the authenticity and integrity of emails and files, confirming they haven't been tampered with.
  • File Encryption: Protects sensitive data by encrypting files, ensuring that only authorized users can access them.
  • Key Management: Manages public and private keys for secure communication and data protection.

1. Describe the different types of cyber threats?

Cyber threats are malicious activities aimed at disrupting, damaging, or gaining unauthorized access to computer systems, networks, or data. Understanding these threats is crucial for developing effective cybersecurity measures. Here, we discuss several prevalent types of cyber threats in detail.

1. Malware:

Malware, short for malicious software, encompasses various harmful programs designed to damage or disrupt systems, steal information, or gain unauthorized access. Common types of malware include:

  • Viruses
  • Worms
  • Trojans
  • Ransomware
  • Spyware

2. Phishing:

Phishing is a form of social engineering where attackers deceive individuals into providing sensitive information, such as login credentials and financial details. Typically, phishing attacks are carried out through:

  • Email Phishing
  • Spear Phishing
  • Smishing and Vishing

3. Denial of Service (DoS):

A Denial of Service attack aims to make a network or service unavailable to its intended users by overwhelming it with a flood of illegitimate requests. Variants include:

  • Distributed Denial of Service (DDoS)
  • Application Layer DoS

4. Man-in-the-Middle (MitM):

MitM attacks occur when an attacker intercepts and possibly alters the communication between two parties without their knowledge. This can lead to:

  • Eavesdropping
  • Data Manipulation

5. SQL Injection:

SQL injection is a code injection technique that exploits vulnerabilities in an application's software by inserting malicious SQL statements into an input field. This can lead to:

  • Data Theft
  • Data Manipulation
  • Access Control Bypass

6. Advanced Persistent Threats (APTs):

APTs are prolonged and targeted cyberattacks where attackers infiltrate a network and remain undetected for an extended period, often to steal sensitive data. Characteristics include:

  • Stealth
  • Persistence
  • Targeted

7. Insider Threats:

Insider threats involve risks posed by individuals within the organization, such as employees, contractors, or business partners. These threats can be categorized into:

  • Malicious Insiders
  • Negligent Insiders

In conclusion, the landscape of cyber threats is diverse and constantly evolving. Organizations must stay vigilant and adopt comprehensive cybersecurity measures to protect against these threats. This includes implementing robust security policies, conducting regular security training, deploying advanced security technologies, and continuously monitoring for potential threats.

2. Brief the various cyber security safeguards.

Cybersecurity safeguards are measures taken to protect computer systems, networks, and data from cyber threats and attacks. These safeguards are essential for maintaining the confidentiality, integrity, and availability of information. Below, we discuss various cybersecurity safeguards in detail.

1. Firewalls:

Firewalls are network security devices that monitor and control incoming and outgoing network traffic based on predetermined security rules. They act as a barrier between trusted internal networks and untrusted external networks. Types of firewalls include:

  • Packet-Filtering Firewalls
  • Stateful Inspection Firewalls
  • Next-Generation Firewalls (NGFWs)

2. Antivirus and Anti-Malware Software:

These programs detect, prevent, and remove malicious software from computer systems. They work by scanning files, identifying known malware signatures, and monitoring system behavior for suspicious activities. Key features include:

  • Real-Time Protection
  • Heuristic Analysis
  • Regular Updates

3. Encryption:

Encryption converts data into a secure format that can only be read by someone with the decryption key. It is essential for protecting sensitive information during storage and transmission. Common encryption methods include:

  • Symmetric Encryption
  • Asymmetric Encryption
  • End-to-End Encryption

4. Access Control:

Access control mechanisms ensure that only authorized users can access specific resources. This involves authentication and authorization. Common methods include:

  • Password-Based Authentication
  • Multi-Factor Authentication (MFA)
  • Role-Based Access Control (RBAC)

5. Regular Updates and Patch Management:

Keeping software and systems up-to-date is crucial for protecting against known vulnerabilities. This involves:

  • Software Updates
  • Patch Management

6. Security Awareness Training:

Educating employees about cybersecurity best practices and potential threats is essential for minimizing human error. Training should cover topics such as:

  • Recognizing Phishing Attacks
  • Safe Internet Practices
  • Incident Reporting

7. Intrusion Detection and Prevention Systems (IDPS):

IDPS monitor network and system activities for malicious behavior and policy violations. They can alert administrators to potential threats and, in some cases, take automatic action to prevent attacks. Types include:

  • Network-Based IDS/IPS (NIDS/NIPS)
  • Host-Based IDS/IPS (HIDS/HIPS)

8. Data Loss Prevention (DLP):

DLP solutions help prevent sensitive data from being lost, stolen, or accidentally shared. They work by monitoring data and enforcing policies to protect data. Key components include:

  • Content Discovery
  • Policy Enforcement
  • Incident Response

9. Backup and Recovery:

Regularly backing up data is essential for ensuring data availability and integrity in case of a cyberattack or system failure. Effective backup and recovery strategies include:

  • Regular Backups
  • Offsite Storage
  • Disaster Recovery Plans

In conclusion, implementing a combination of these cybersecurity safeguards can significantly enhance an organization's security posture. It is important to adopt a layered security approach, continuously monitor for threats, and adapt to the evolving cybersecurity landscape to protect against cyber threats effectively.

3. Describe the basic security for SOAP Services.

SOAP (Simple Object Access Protocol) is a protocol for exchanging structured information in web services. Ensuring the security of SOAP services is crucial to protect sensitive data and maintain the integrity of communication. Here, we discuss the basic security measures for SOAP services in detail.

1. Transport Layer Security (TLS):

TLS is used to secure communication between the client and the server by encrypting the data transmitted over the network. Key features include:

  • Encryption
  • Integrity
  • Authentication

2. WS-Security:

WS-Security is a set of protocols that apply security to SOAP messages by providing message integrity, confidentiality, and authentication. It includes:

  • XML Signature
  • XML Encryption
  • Security Tokens

3. Access Control:

Access control mechanisms ensure that only authorized users can access the SOAP services. This involves:

  • Authentication
  • Authorization

4. Input Validation:

Validating the data sent to the SOAP service is essential to prevent injection attacks and ensure data integrity. This involves:

  • Sanitizing Input
  • Schema Validation

5. Security Tokens:

Security tokens are used to secure SOAP messages by providing a means of authentication and ensuring that messages are sent by trusted sources. Common token types include:

  • UsernameToken
  • BinarySecurityToken
  • SAML Tokens

6. Message-Level Security:

In addition to transport-level security (TLS), message-level security ensures that each SOAP message is individually secured, regardless of the transport mechanism. This includes:

  • Signature
  • Encryption

7. Security Policy Enforcement:

Defining and enforcing security policies for SOAP services ensures that all security requirements are consistently applied. This can be achieved through:

  • WS-Policy
  • Policy Assertions

8. Logging and Monitoring:

Regularly logging and monitoring SOAP service activities help detect and respond to security incidents. Key practices include:

  • Audit Logs
  • Intrusion Detection Systems (IDS)
  • Alerts and Notifications

9. Secure Coding Practices:

Implementing secure coding practices when developing SOAP services reduces the risk of vulnerabilities. This involves:

  • Avoiding Hard-Coded Credentials
  • Error Handling
  • Regular Code Reviews

In conclusion, securing SOAP services involves a combination of transport-level security, message-level security, access control, input validation, and secure coding practices. By implementing these measures, organizations can protect their web services from various cyber threats and ensure the confidentiality, integrity, and availability of their data.

4. Brief the different types of firewalls.

Firewalls are essential components of network security, designed to monitor and control incoming and outgoing traffic based on predetermined security rules. They act as a barrier between trusted and untrusted networks, such as the internet. Here, we discuss the different types of firewalls in detail.

1. Packet-Filtering Firewalls:

Packet-filtering firewalls operate at the network layer (Layer 3) of the OSI model and make decisions based on individual packets' headers. They inspect the source and destination IP addresses, port numbers, and protocol types to determine whether to allow or block traffic.

2. Stateful Inspection Firewalls:

Stateful inspection firewalls, also known as dynamic packet-filtering firewalls, monitor the state of active connections and make decisions based on the context of the traffic. They operate at both the network layer and transport layer (Layer 4).

3. Application-Level Gateways (Proxy Firewalls):

Application-level gateways, or proxy firewalls, operate at the application layer (Layer 7) of the OSI model. They act as intermediaries between clients and servers, inspecting the entire content of messages for specific applications.

4. Next-Generation Firewalls (NGFWs):

Next-generation firewalls combine traditional firewall capabilities with advanced features like deep packet inspection, intrusion prevention systems (IPS), and application awareness. They operate across multiple layers of the OSI model.

5. Network Address Translation (NAT) Firewalls:

NAT firewalls hide the internal IP addresses of a network by translating them into a single public IP address. This provides an additional layer of security by masking the internal network structure from external parties.

6. Cloud Firewalls:

Cloud firewalls, also known as firewall-as-a-service (FaaS), are deployed in the cloud and designed to protect cloud infrastructure and services. They can operate at various layers depending on the service model.

7. Virtual Firewalls:

Virtual firewalls are software-based firewalls deployed within virtualized environments, such as virtual machines (VMs) or virtual private clouds (VPCs). They provide the same functionality as physical firewalls but are optimized for virtual environments.

In conclusion, different types of firewalls offer varying levels of security, performance, and complexity. Organizations must choose the appropriate type of firewall based on their specific needs, network architecture, and threat landscape. Combining multiple firewall types in a layered security approach can provide comprehensive protection against a wide range of cyber threats.
